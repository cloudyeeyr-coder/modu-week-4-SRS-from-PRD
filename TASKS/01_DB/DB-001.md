## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: 로컬 개발 환경(SQLite) 연결 검증
- Given: `.env.local` 파일에 `DATABASE_URL`이 로컬 SQLite 경로(예: `file:./dev.db`)로 설정됨
- When: `npx prisma db push` 또는 `npx prisma migrate dev` 명령어를 실행함
- Then: 로컬에 `dev.db` 파일이 생성되며, 성공적으로 스키마가 동기화되었다는 정상 종료 메시지가 출력된다.

Scenario 2: ORM 엔진 포맷팅 검증
- Given: 초기화된 빈 `schema.prisma` 구조
- When: `npx prisma format`을 실행함
- Then: Prisma 파일이 오류 없이 정상 포맷팅된다.

## :gear: Technical & Non-Functional Constraints
- 안정성: Vercel Serverless 환경과 Supabase를 연결할 때 커넥션 고갈을 방지하기 위해 Connection Pooling 설정(`?pgbouncer=true` 파라미터 등) 여부를 검토해야 함.
- 보안: `DATABASE_URL`이 포함된 `.env` 시리즈 파일들은 절대 Git 레포지토리에 커밋되지 않도록 `.gitignore` 적용 필수.
