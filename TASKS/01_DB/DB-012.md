## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: 수기 견적 리드 접수 저장
- Given: 사이트에서 폼을 제출한 사용자
- When: DB에 폼 데이터(모델, 수량, 개월수)를 Insert함
- Then: Enum의 기본값이 `pending`으로 처리되어 문제없이 신규 로우가 생성된다.

Scenario 2: 어드민 응답 데이터 처리
- Given: 대시보드에서 `responded`로 견적 상태를 바꾸고자 하는 운영팀
- When: `responseData` JSON 필드에 견적서 금액 등의 객체 묶음을 UPDATE함
- Then: 오류 없이 JSON 스키마 필드에서 데이터가 받아들여지고 상태 변경이 커밋된다.

## :gear: Technical & Non-Functional Constraints
