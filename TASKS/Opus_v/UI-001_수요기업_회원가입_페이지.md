## :dart: Summary
- 기능명: [UI-001] 수요기업(Buyer) 회원가입 페이지
- 목적: 로봇 도입을 희망하는 수요기업(SME)이 플랫폼에 최초 진입하는 온보딩 관문. 직관적인 다단계 폼 UI를 제공하여 필수 기업 정보(회사명, 사업자등록번호, 지역, 담당자)를 수집하고, Zod 기반 인라인 유효성 검증으로 잘못된 입력을 즉시 피드백하여 가입 이탈률을 최소화한다. 가입 완료 시 `signup_complete` 이벤트를 DB 이벤트 로그에 기록하고, SI 파트너 검색 페이지(`/search`)로 리다이렉트한다.

## :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: 정상적인 수요기업 회원가입**
- **Given:** 신규 수요기업 담당자가 `/signup/buyer` 페이지에 접근함
- **When:** 필수 정보(회사명: "테스트기업", 사업자등록번호: "123-45-67890", 지역: "서울", 세그먼트: "Q1", 담당자명: "홍길동", 이메일: "test@example.com", 전화번호: "010-1234-5678")를 입력하고 '가입하기' 버튼을 클릭함
- **Then:** `signupBuyer` Server Action이 호출되고, 성공 시 `signup_complete` 이벤트가 DB 이벤트 로그에 기록되며, `/search` 페이지로 리다이렉트됨

**Scenario 2: 중복 사업자등록번호로 가입 시도**
- **Given:** 이미 DB에 존재하는 사업자등록번호 `"123-45-67890"`이 주어짐
- **When:** 해당 사업자등록번호로 회원가입을 제출함
- **Then:** 페이지가 크래시되지 않고, 사업자등록번호 Input 필드 하단에 `"이미 가입된 사업자등록번호입니다"` 인라인 에러 메시지가 표시됨

**Scenario 3: 필수 필드 누락 시 인라인 유효성 에러 표시**
- **Given:** 사용자가 가입 폼에서 회사명 필드를 비워 둠
- **When:** '가입하기' 버튼을 클릭하거나 다른 필드로 포커스를 이동함
- **Then:** 회사명 필드 하단에 `"회사명을 입력해주세요"` 인라인 에러 메시지가 즉시 표시되고, 폼 제출이 차단됨

**Scenario 4: 사업자등록번호 포맷 오류**
- **Given:** 사용자가 사업자등록번호 필드에 `"12345"` (잘못된 포맷)을 입력함
- **When:** 입력 중 실시간 유효성 검증이 트리거됨
- **Then:** 사업자등록번호 필드 하단에 `"올바른 사업자등록번호 형식(XXX-XX-XXXXX)을 입력해주세요"` 인라인 에러 메시지가 표시됨

**Scenario 5: 반응형 레이아웃 (Mobile)**
- **Given:** 사용자가 모바일 기기(화면 너비 ≤ 768px)로 `/signup/buyer`에 접근함
- **When:** 페이지가 렌더링됨
- **Then:** 폼이 단일 컬럼 풀 너비로 표시되며, 화면 밖으로 콘텐츠가 벗어나지 않고, 모든 입력 필드가 터치 가능한 크기(최소 44px 높이)를 가짐

**Scenario 6: 제출 중 로딩 상태**
- **Given:** 사용자가 모든 필수 정보를 정상적으로 입력함
- **When:** '가입하기' 버튼을 클릭함
- **Then:** 버튼이 비활성화(disabled)되고 '계정 생성 중...' 텍스트와 로딩 스피너가 표시되어, 중복 제출이 방지됨

## :gear: Technical & Non-Functional Constraints

### UI/스타일링
- **프레임워크:** Next.js App Router + React Server Components (페이지 셸) / Client Component (폼 인터랙션)
- **UI 라이브러리:** Tailwind CSS + shadcn/ui 컴포넌트 (`Card`, `Input`, `Label`, `Button`, `Select`) — CON-14 준수
- **유효성 검증:** Zod + react-hook-form (`@hookform/resolvers/zod`)
- **상태 관리:** `useFormState` / `useTransition` (Server Action 연동)

### 성능
- 페이지 LCP p95 ≤ 2,000ms 달성 — REQ-NF-001
- 인라인 유효성 에러 표시 ≤ 200ms (클라이언트 사이드 즉시 반응)
- 폼 제출 → 리다이렉트 완료 p95 ≤ 3,000ms

### 안정성
- 서버 에러(409/400/500) 시 페이지 크래시 없이 Graceful Error Handling 필수
- 네트워크 오류 시 재시도 가능 상태로 폼 데이터 유지 (폼 초기화 방지)

### 보안
- 사업자등록번호, 이메일, 전화번호 등 개인정보는 클라이언트 콘솔/로그에 노출 금지
- XSS 방지: 모든 사용자 입력 값 이스케이프 처리 (React 기본 동작 + Zod 검증)
- CSRF 보호: Server Action 기본 CSRF 토큰 검증 활용

### 접근성 (Accessibility)
- 모든 Input에 연결된 `<Label>` 태그 필수
- 에러 메시지는 `role="alert"` 또는 `aria-live="polite"`로 스크린 리더 접근 보장
- 색상 대비 WCAG 2.1 AA 기준 충족 (에러 메시지 빨간색 + 텍스트 라벨 조합)
