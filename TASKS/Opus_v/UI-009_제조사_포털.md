## :dart: Summary
- 기능명: [UI-009] 제조사 포털 (Manufacturer Portal)
- 목적: 로봇 제조사가 SI 파트너에 대한 인증 뱃지 발급/철회, 파트너십 제안 발송, 현재 파트너 현황을 관리하는 전용 포털을 제공한다. Brand-Agnostic 개방형 구조(CON-03)를 기반으로 3개 이상의 제조사가 독립적으로 뱃지를 운영할 수 있으며, 파트너 제안의 수락/거절/만료까지 전체 라이프사이클을 시각화한다.

## :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: 뱃지 발급**
- **Given:** 제조사가 SI "로봇시공"에 뱃지 발급
- **When:** SI 선택 + 만료일 설정 + '발급' 확인
- **Then:** BADGE INSERT (is_active=true), SI 프로필 반영 ≤1시간

**Scenario 2: 뱃지 철회**
- **Given:** 활성 뱃지 보유 SI
- **When:** '철회' → 사유 입력 → 확인
- **Then:** is_active=false, revoked_at 기록, SI 프로필 비노출 ≤10분

**Scenario 3: 파트너 제안 발송**
- **Given:** 뱃지 미보유 SI 선택
- **When:** '파트너 제안 발송' 클릭
- **Then:** PROPOSAL INSERT (status=pending, deadline=D+5), SI 알림 발송 ≤3초

**Scenario 4: 제안 만료 후 대안 SI 추천**
- **Given:** SI 미응답 5영업일 경과 (status=expired)
- **When:** 제안 목록 조회
- **Then:** 만료 표시 + "대안 SI 3개사" 추천 섹션 표시

**Scenario 5: 파트너 현황 대시보드**
- **Given:** 활성 파트너 5사, 대기 제안 3건, 만료 예정 2건
- **When:** 대시보드 접근
- **Then:** KPI 카드에 5/3/2 정확히 표시, 파트너 목록 정상 로드

**Scenario 6: 중복 뱃지 발급 차단**
- **Given:** SI에 이미 활성 뱃지 존재
- **When:** 동일 SI에 뱃지 발급 시도
- **Then:** "이미 활성 뱃지가 존재합니다" 에러 메시지

- **성능:** 대시보드 LCP p95 ≤ 2,000ms, 뱃지 반영 ≤1시간, 비노출 ≤10분

- **접근성:** Combobox/DatePicker 키보드 접근, WCAG 2.1 AA
