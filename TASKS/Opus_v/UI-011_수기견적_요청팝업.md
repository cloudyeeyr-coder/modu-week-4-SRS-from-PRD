## :dart: Summary
- 기능명: [UI-011] 수기 견적 요청 팝업
- 목적: RaaS 비용 비교 계산기(UI-010)에서 특정 플랜을 선택한 수요기업이 "운영팀에 맞춤 견적 요청하기" 폼을 통해 연락처와 요청 정보를 제출하는 팝업 UI를 제공한다. 제출 완료 시 QUOTE_LEAD가 DB에 저장되고, Admin에게 Slack+이메일 알림이 발송되며, 사용자에게 "요청 완료! 운영팀이 2영업일 내 연락드립니다" 확인 화면을 표시한다.

- [ ] shadcn/ui `Dialog` 기반 모달 팝업 컴포넌트 구현
- [ ] UI-010 계산기 결과에서 '이 플랜으로 견적 요청' 버튼 클릭 시 모달 오픈
- [ ] 모달 헤더: "운영팀에 맞춤 견적 요청하기"
- [ ] 계산기에서 선택한 정보 자동 프리필: 로봇 모델, 수량, 기간

### 2단계: 견적 요청 폼 필드
- [ ] 로봇 모델 (`robot_model`) — `Input` (프리필, 수정 가능)
- [ ] 수량 (`quantity`) — `Input[type=number]` (프리필, 수정 가능)
- [ ] 희망 계약 기간 (`term_months`) — `Select` (프리필, 수정 가능)
- [ ] 담당자 이름 (`contact_name`) — `Input` (필수, 로그인 시 자동 채움)
- [ ] 담당자 이메일 (`contact_email`) — `Input` (필수, 로그인 시 자동 채움)
- [ ] 담당자 전화번호 (`contact_phone`) — `Input` (필수, 자동 하이픈)
- [ ] 추가 요청 사항 — `Textarea` (선택, 최대 500자)

### 3단계: 유효성 검증
- [ ] Zod 스키마 `quoteRequestSchema`:
  - `robot_model`: `z.string().min(1)` — 필수
  - `quantity`: `z.number().int().min(1)` — 1 이상
  - `term_months`: `z.number().int().min(1)` — 1 이상
  - `contact_name`: `z.string().min(1).max(100)` — 필수
  - `contact_email`: `z.string().email()` — 이메일 포맷
  - `contact_phone`: `z.string().regex(/^01[016789]-\d{3,4}-\d{4}$/)` — 전화번호 포맷
- [ ] 인라인 에러 메시지 (react-hook-form + Zod)

### 4단계: 서버 통신 및 완료 화면
- [ ] '견적 요청하기' 버튼 → `FC-022` (`requestManualQuote`) Server Action 호출
- [ ] 제출 중 '요청 중...' 로딩 스피너 + 버튼 disabled

  - '확인' 버튼 → 모달 닫기
- [ ] 에러 핸들링: 400/500 → Toast "요청에 실패했습니다. 다시 시도해주세요."

## :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: 정상 견적 요청 제출**
- **Given:** 계산기에서 "UR10e / 5대 / 36개월 / RaaS" 선택 후 팝업 오픈
- **When:** 연락처 입력 → '견적 요청하기' 클릭
- **Then:** QUOTE_LEAD INSERT (status=pending), Admin Slack+이메일 알림, "요청 완료" 확인 화면 표시

**Scenario 2: 프리필 데이터 자동 채움**
- **Given:** 계산기에서 모델/수량/기간 선택 후 팝업 오픈
- **When:** 모달 렌더링
- **Then:** 로봇 모델, 수량, 기간 필드에 선택값이 자동 프리필됨

**Scenario 3: 필수 필드 누락 제출 차단**
- **Given:** 담당자 이메일 미입력
- **When:** '견적 요청하기' 클릭
- **Then:** "이메일을 입력해주세요" 인라인 에러, 제출 차단

**Scenario 4: 요청 완료 확인 화면**
- **Given:** 견적 요청 성공
- **When:** 완료 화면 렌더링
- **Then:** "운영팀이 2영업일 내 연락드립니다" + 요청 번호 표시

**Scenario 5: 비로그인 사용자 안내**
- **Given:** 비로그인 상태에서 견적 요청 시도
- **When:** '견적 요청' 버튼 클릭
- **Then:** "견적 요청을 위해 로그인이 필요합니다" 안내 → 로그인 페이지
