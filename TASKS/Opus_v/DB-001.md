## :dart: Summary
- 기능명: [DB-001] Prisma ORM 초기 설정 및 SQLite/PostgreSQL 이중 환경 구성
- 목적: 전체 데이터베이스 스키마의 **단일 진실 공급원(SSOT)** 역할을 하는 Prisma ORM을 초기화하고, 로컬 개발 환경(SQLite)과 프로덕션 배포 환경(Supabase PostgreSQL)에서 동일한 `schema.prisma`로 동작할 수 있는 이중 데이터소스 기반을 확립한다. 이 태스크는 모든 DB-00x 테이블 마이그레이션 및 Seed 스크립트의 기반이 되는 **프로젝트 데이터 레이어의 최초 설정** 태스크이다.

## :white_check_mark: Task Breakdown (실행 계획)

### Phase A: Prisma 패키지 설치 및 초기화
- [ ] `prisma` (DevDependency) 및 `@prisma/client` (Runtime Dependency) 패키지 설치
  ```bash
  npm install -D prisma
  npm install @prisma/client
  ```
- [ ] `npx prisma init` 실행하여 `prisma/schema.prisma` 파일 및 `.env` 기본 구조 생성

### Phase B: `schema.prisma` 기본 설정
- [ ] `datasource db` 블록 설정 — `provider = "postgresql"` (배포 기준), `url = env("DATABASE_URL")`
  - SQLite 로컬 개발 시 `DATABASE_URL=file:./dev.db`로 전환 가능하도록 설계
  - **Prisma 5.x 이상** `datasource` provider 동적 전환 제약 사항 문서화
- [ ] `generator client` 블록 설정 — `provider = "prisma-client-js"`, `previewFeatures` 필요 시 명시
- [ ] `schema.prisma` 내 후속 엔티티(DB-002~DB-017) 추가를 위한 모델 배치 컨벤션 주석 작성 (도메인별 영역 구분)

### Phase C: 환경 변수 및 보안 설정
- [ ] `.env.example` 파일 생성 — `DATABASE_URL`, `DIRECT_URL` 템플릿 제공
  ```dotenv
  # 로컬 개발 (SQLite)
  DATABASE_URL="file:./dev.db"

  # 배포 환경 (Supabase PostgreSQL) — 실제 값은 Vercel Dashboard에서 관리
  # DATABASE_URL="postgresql://[user]:[password]@[host]:[port]/[db]?pgbouncer=true"
  # DIRECT_URL="postgresql://[user]:[password]@[host]:[port]/[db]"
  ```
- [ ] `.env.local` 파일 생성 — 로컬 SQLite 기본 URL 설정 (`file:./dev.db`)
- [ ] `.gitignore` 확인 — `.env`, `.env.local`, `.env.production.local`, `prisma/dev.db`, `prisma/dev.db-journal` 항목 포함 여부 검증

### Phase D: Connection Pooling 및 배포 환경 전략 수립
- [ ] Supabase PostgreSQL Connection Pooling 전략 문서화
  - Vercel Serverless 환경에서의 커넥션 고갈 방지를 위한 `pgbouncer=true` 파라미터 적용 방안
  - `DATABASE_URL` (Pooling Mode, Transaction Mode) vs `DIRECT_URL` (Direct Connection, Migration 전용) 이중 URL 전략
- [ ] Prisma Client 인스턴스 싱글턴 패턴 구현 (`lib/prisma.ts`)
  ```typescript
  // lib/prisma.ts
  import { PrismaClient } from '@prisma/client';

  const globalForPrisma = globalThis as unknown as { prisma: PrismaClient };

  export const prisma = globalForPrisma.prisma ?? new PrismaClient();

  if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma;
  ```

## :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: 로컬 개발 환경(SQLite) 연결 및 스키마 동기화 검증**
- Given: `.env.local` 파일에 `DATABASE_URL`이 로컬 SQLite 경로(`file:./dev.db`)로 설정됨
- When: `npx prisma db push` 명령어를 실행함
- Then: 프로젝트 루트의 `prisma/` 디렉토리에 `dev.db` 파일이 정상 생성되며, 콘솔에 "Your database is now in sync with your Prisma schema" 성공 메시지가 출력된다. Exit code = 0

**Scenario 2: Prisma Schema 포맷팅 및 유효성 검증**
- Given: `schema.prisma` 파일에 `datasource db`, `generator client` 블록이 작성됨
- When: `npx prisma format` 명령어를 실행함
- Then: 포맷팅이 오류 없이 완료되고, Prisma 파일이 정상 구조로 정렬된다. Exit code = 0

**Scenario 3: Prisma Client 생성 및 import 가능 여부 검증**
- Given: `schema.prisma` 파일이 유효한 상태이고, `@prisma/client` 패키지가 설치됨
- When: `npx prisma generate` 명령어를 실행함
- Then: `node_modules/.prisma/client`에 타입 정의 파일이 생성되고, `import { PrismaClient } from '@prisma/client'`가 TypeScript 타입 에러 없이 정상 인식된다

**Scenario 4: Prisma Client 싱글턴 인스턴스 정상 동작**
- Given: `lib/prisma.ts`에 싱글턴 패턴이 구현됨
- When: Next.js dev 서버(`npm run dev`)가 HMR(Hot Module Replacement) 리로드를 수행함
- Then: 콘솔에 "warn(prisma-client) Already N Prisma Clients are actively running" 경고가 출력되지 않는다 (커넥션 누수 방지 확인)

**Scenario 5: .gitignore 보안 검증**
- Given: `.env.local` 파일에 `DATABASE_URL` 환경 변수가 설정됨
- When: `git status` 명령어를 실행함
- Then: `.env.local`, `.env`, `prisma/dev.db`, `prisma/dev.db-journal` 파일이 Untracked/Staged 목록에 포함되지 않는다

## :gear: Technical & Non-Functional Constraints

- **호환성 (CON-13):** 단일 `schema.prisma`로 SQLite(로컬) / PostgreSQL(배포) 이중 운영. 후속 스키마(DB-002~017)에서 Prisma 타입 매핑 규칙 (`Json`, `Decimal`, `enum` 등)을 일관되게 적용할 수 있도록 ORM 매핑 노트를 `schema.prisma` 상단에 주석으로 기재한다.
- **안정성 (Serverless 환경):** Vercel Serverless Function의 Cold Start 및 동시 실행 특성 상 커넥션 고갈을 방지하기 위해 반드시 **Prisma Client 싱글턴 패턴** (`lib/prisma.ts`)을 적용한다. Supabase 연동 시 **Connection Pooling** (`?pgbouncer=true`) 사용을 권장한다.
- **보안:** `DATABASE_URL` 등 실제 인증 정보가 포함된 `.env*` 파일은 절대 Git 레포지토리에 커밋 금지. `.gitignore`에 명시적으로 등록 필수.
- **성능:** Prisma Client 생성 시 `log: ['query']` 옵션은 로컬 개발 환경에서만 활성화하고, 프로덕션에서는 비활성화하여 불필요한 I/O 오버헤드를 방지한다.
