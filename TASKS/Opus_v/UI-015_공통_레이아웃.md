## :dart: Summary
- 기능명: [UI-015] 공통 레이아웃 (Global Layout)
- 목적: 플랫폼의 모든 페이지에 공통으로 적용되는 반응형 헤더, 사이드바, 푸터 레이아웃과 역할별(Buyer/SI Partner/Manufacturer/Admin) 네비게이션 메뉴를 구현한다. 사용자의 인증 상태와 역할에 따라 노출 메뉴가 동적으로 변경되며, 모든 UI 태스크(UI-001~014)의 기반 프레임이 된다.

- [ ] **비로그인 상태**: 로고 + '로그인' + '회원가입(수요기업/SI파트너)' 버튼

- [ ] 반응형: Desktop(가로 네비게이션) → Mobile(햄버거 메뉴 + 모바일 드로어)

### 4단계: 사이드바 (포털별)
- [ ] Admin/Manufacturer/SI 포털은 좌측 사이드바 레이아웃
- [ ] 사이드바 항목: 아이콘 + 텍스트 라벨

- [ ] Tablet: 접이식 (아이콘만 표시, 호버 시 텍스트)

## :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: Buyer 역할 네비게이션**
- **Given:** Buyer 역할로 로그인
- **When:** 헤더 렌더링
- **Then:** SI 검색, 계산기, 내 계약, AS 내역, O2O 예약 메뉴 표시

**Scenario 2: Admin 사이드바 표시**
- **Given:** Admin 역할로 로그인
- **When:** `/admin` 접근
- **Then:** 좌측 사이드바에 대시보드/에스크로/AS SLA/이벤트/분쟁 메뉴 표시

**Scenario 3: 비로그인 헤더**
- **Given:** 비로그인 상태
- **When:** 페이지 접근
- **Then:** 로고 + '로그인' + '회원가입' 버튼만 표시

**Scenario 4: 반응형 Mobile 네비게이션**
- **Given:** 모바일 (≤768px) 접근
- **When:** 헤더 렌더링
- **Then:** 햄버거 메뉴 아이콘 표시, 클릭 시 모바일 드로어 전개

**Scenario 5: 역할 불일치 접근 차단**
- **Given:** SI Partner 역할로 로그인
- **When:** `/admin` 접근 시도
- **Then:** 403 Forbidden 페이지 표시

**Scenario 6: 현재 페이지 하이라이트**
- **Given:** Buyer가 `/search` 페이지 접근
- **When:** 네비게이션 렌더링
- **Then:** "SI 파트너 검색" 메뉴 항목 활성 하이라이트 + `aria-current="page"`

## :gear: Technical & Non-Functional Constraints
- **구현:** Server Component (세션 확인, 역할 조회) + Client Component (모바일 드로어, 드롭다운)
- **UI:** Tailwind CSS + shadcn/ui (`NavigationMenu`, `DropdownMenu`, `Sheet`, `Avatar`) — CON-14
- **폰트:** Inter + Pretendard (한국어) — Google Fonts CDN
- **인증:** NextAuth.js / Supabase Auth 세션 기반, RBAC 4역할 (REQ-NF-016)
- **성능:** 레이아웃 렌더링이 페이지 LCP에 영향 최소화 (Partial Prerendering 활용)
- **보안:** 세션 토큰 HttpOnly Cookie, CSRF 보호
- **접근성:** 스킵 네비게이션, 시맨틱 nav, aria 속성, WCAG 2.1 AA
