## :dart: Summary
- 기능명: [FC-017] 제조사에 의한 뱃지 철회(`revokeBadge`) 트랜잭션
- 목적: 계약 만료나 신뢰도 문제로 인해 제조사가 뱃지를 회수할 때, 해당 레코드를 삭제(Delete)하지 않고 상태를 무효화(Update)시켜 검색 필터(FQ-003)에서 100% 누락되도록 하는 강제 차단 액션.

## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: 데이터 보존주의 원칙 (Soft Delete 룰)
- Given: 철회 버튼 액션
- When: FC 계층이 쿼리를 실행함
- Then: 이 로직은 `prisma.badge.delete` 를 돌리지 않는다. 단지 `update` 만 날려 이력을 보존해야 감사(Audit) 관점에 유효하다는 이 명세의 지시를 따름.

Scenario 2: NFR 즉각 제거
- Given: 캐싱된 SI 검색 목록 화면
- When: 뱃지가 철회됨
- Then: 이 FC의 끝단에서 터뜨린 캐시 초기화 명령으로 인해, 1분 이상 지연되지 않고 10분 내(NFR)는 물론, 페이지 리로드 시 해당 SI의 뱃지가 사라짐이 확인된다.
