## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: 계약 금액 검증
- Given: `totalAmount`가 0 이하로 설정되거나 소수점이 설계 한계를 초과하는 페이로드
- When: 클라이언트가 액션을 호출함
- Then: Zod 레벨에서 즉각 반려되어 비정상적 금액 계약 생성을 막는다.

Scenario 2: 안내(가상계좌) 정보 반환
- Given: 올바른 계약 데이터
- When: 계약이 성공적으로 생성되었다고 시뮬레이션함(Response)
- Then: 반환되는 데이터 모델 안에는 입금을 유도할 대상 법인 계좌 번호나 텍스트가 포함되어 있어야 UI(UI-005)가 이를 랜더링 할 수 있다.
