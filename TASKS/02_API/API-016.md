## :white_check_mark: Task Breakdown (실행 계획)
- [ ] `IssueBadgeSchema` (Zod) 작성: 
  - 필수 파라미터: `manufacturerId` (발급자), `siPartnerId` (수신자)
  - 선택/필수 파라미터: `durationMonths` (만료 기간 결정을 위한 숫자, 예: 12) 선언
- [ ] 인터페이스 주석 명세: 이 액션은 내부적으로 DB에 Insert할 때 기본적으로 `isActive: true` 와 `expiresAt = Date.now() + durationMonths`의 파생 처리를 낳는다는 것을 DTO 수준에서 가이드한다.
- [ ] (보안 검증) 요청자의 Session ID가 `manufacturerId` 와 일치하여야 한다는 Action 통제 원칙 문서화.
- [ ] 결과 반환: `{ success, data: BadgeModel }`
- [ ] 오류 코드 모음 (`BADGE_ALREADY_ISSUED`, `FORBIDDEN_MANUFACTURER`)

## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: 권한 통제 명세 적용
- Given: 제조사 A의 세션 정보
- When: 타 제조사 B의 `manufacturerId` 파라미터로 이 DTO를 사용하여 호출하려 함
- Then: 이 명세의 보안 제약사항에 따라 `FORBIDDEN_MANUFACTURER` 에러 메시지가 리턴됨을 코드 레벨 단위 테스트(추후 작성)에서 보장 받을 수 있도록 타입 설계됨.

Scenario 2: 만료일 데이터 체계
- Given: 정상 파트너와 기간 12개월(duration)
- When: 서버가 응답값을 Return함
- Then: 반환되는 바디 내에 명시적인 `expiresAt` 날짜 포맷이 담겨 있어서, 프론트에서 즉시 "2027년까지 유효합니다"를 렌더링 할 수 있는 구조이다.

## :gear: Technical & Non-Functional Constraints
