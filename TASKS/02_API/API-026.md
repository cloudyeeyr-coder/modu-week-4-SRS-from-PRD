## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: 슬랙 포맷 파싱 및 결합
- Given: "에스크로 결제 오류율 급증(현재 0.8%)" 이라는 시스템 장애 판단
- When: 백엔드 로직이 `SlackMessagePayload` 형식에 이 데이터를 태워 넘김
- Then: 포맷이 Block Kit 의 JSON 객체( `blocks`, `text` 형태 )로 자연스럽게 파싱됨이 TS 타입 선언 에러 없이 이루어져야 한다.

Scenario 2: 모듈화 호출 인터페이스
- Given: AS 미배정 크론 스크립트 작성 중
- When: 이 인터페이스 서명(Signature)을 이용해 함수를 호출함
- Then: 이 인터페이스는 단일화 되어 있어, 어떤 모니터링 이벤트라도 타겟을 쉽게 지정해 함수처럼 쓸 수 있게 가이드 되어 있다.
