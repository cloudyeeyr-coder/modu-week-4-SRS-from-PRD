## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: 비정상적 수량 방어 설계
- Given: 웹에서 실수로 수량(Quantity) 파라미터에 `0` 혹은 음수가 입력됨
- When: 계산기를 호출함
- Then: API 파서 지점에서 Zod 제약(`min(1)`)이 발동, 백엔드 계산 로직으로 넘어가기 이전에 `INVALID_QUANTITY` 형식을 즉시 튕겨낸다.

Scenario 2: DTO 반환 맵
- Given: 정상적인 로봇 모델 "RBT-123"과 수량 2, 기간 24를 송신함
- When: 계산 성공
- Then: 이 규격 문서가 지정한 `RaasCalculationResponse` 타입과 일치하는 JSON 객체가 일시불(Opex), 리스, RaaS 월별 가격을 명확히 반환하여 차트 UI 렌더링(UI-010)을 돕는다.
