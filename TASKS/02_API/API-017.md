## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: 올바른 철회 시나리오
- Given: 활성화된 뱃지 ID
- When: 제조사가 이를 취소하는 파라미터를 보냄
- Then: DTO 타입 규격에 의해 정의된 대로 서버 액션이 리턴되고, 응답에는 즉각 반영을 시사하는 `{ isActive: false }` 정보가 들어 있다.

Scenario 2: 기 비활성 뱃지 재호출 방어
- Given: 이미 만료되거나 취소된 뱃지 ID
- When: 서버 액션을 다시 호출함
- Then: DTO 및 에러 명세에 의해 `BADGE_ALREADY_REVOKED` 에러 코드를 구조화하여 프론트에 뿌려줌.
