- [ ] 인증 실패, 권한 부족 시 응답할 시스템 표준 예외 핸들링 에러 객체 맵핑(`401 Unauthorized`, `403 Forbidden`). 

## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: 세션 타입 확장
- Given: Server Component 나 Client Component 에서 `useSession` 또는 `auth()` 를 호출함
- When: 반환된 세션 객체를 TypeScript 코드로 접근함
- Then: 기본 `session.user.email` 만 있는 것이 아니라 `session.user.role` 이 TS 에러 없이 자동완성 및 제공되어야 한다.

Scenario 2: 권한 매핑 인터페이스 확인
- Given: 미들웨어에서 라우트 가딩(Route Guard)을 작성중인 코드
- When: `session.role` 이 `'admin'` 인지 체크하는 로직을 삽입함
- Then: 이 API 명세/타입 확장 정의에 의해 사전에 정의되지 않은 역할 문자열('guest' 등)을 비교하려 하면 TS 문법 타임에 오류를 내뿜어 개발 실수를 막아준다.
