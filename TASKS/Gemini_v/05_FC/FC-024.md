## :white_check_mark: Task Breakdown (실행 계획)
- [ ] `Supabase Storage` 연결/설정 모듈 셋업 (Supabase-js).

  - 보안 파일 사이즈 체커 (예: 최대 10MB) 및 확장자 제한(`pdf, png, jpg`) 도입.

## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: 확장자 통제 명세 가드
- Given: 해커가 웹 브라우저 단의 필터를 우회하여 `.sh`나 `.php` 등의 악성 파일을 `multipart/form-data` 로 넘김
- When: 이 FC 핸들러가 파싱함
- Then: 저장소로 넘기기 사전에 백엔드 Mime Type 검사에 걸러져 `INVALID_FILE_TYPE` 400 에러를 유발하며 원천 차단해야 함.
