## :dart: Summary
- 기능명: [FC-011] AS 티켓 생성 및 상황 기반 우선순위 조정 로직
- 목적: 접수된 AS 정보를 DB에 저장할 뿐만 아니라, 얽혀있는 SI 파트너가 부도(혹은 계정 비활성) 상태일 경우 관리자가 최우선 개입할 수 있도록 티켓의 `priority` 를 강제로 격상시키는 조건부 생성 파이프라인.

## :white_check_mark: Task Breakdown (실행 계획)
- [ ] 입력 파라미터(증상, 계약ID 등) 유효성 수신

- [ ] 티켓 상태 `reported` 로 인서트 저장.
- [ ] 신규 티켓 생성 알림 이벤트를 Admin 대시보드(혹은 Slack, API-026) 쪽에 Emit 시킴.

## :test_test: Acceptance Criteria (BDD/GWT)
Scenario 1: 부도 기업 징벌적 SLA 강제
- Given: 계약자(SI)가 이미 MOCK-003 단위에서 파트너십이 끊긴 기업
- When: 수요 기업 폼(Client) 단에서 고의/실수로 덜 급하다고 `priority: normal` 로 티켓을 발사함
- Then: 이 FC 계층에서 DB 쿼리 후 SI의 상태를 감지, `urgent` 등급으로 오버라이드(Override) 한 뒤 DB에 저장하여 SLA 가혹 기준을 적용한다.
