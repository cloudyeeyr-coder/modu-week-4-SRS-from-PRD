## :dart: Summary

## :white_check_mark: Task Breakdown (실행 계획)
- [ ] Page Props 내지 서치 파라미터를 파싱할 Zod 스키마 작성 (`SearchPartnerParams`)
  - `page`, `limit` (페이징, 숫자 변환)
  - `region`, `brand`, `badge`, `tier` (단일 또는 배열로 캐스팅)
  - `tags` (쉼표 구분 문자열 파싱 규칙 적용)
- [ ] 뱃지 소유 여부만 걸러낼 Boolean 필드(`hasBadge: boolean`) 규격 추가
- [ ] Response Payload: `{ items: [], totalCount: number, totalPages: number }` 와 같이 페이지네이션을 지원하는 형태의 구조체 타입 정의.

## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: 쿼리 변환 유효성 
- Given: Next.js Server Component에 진입한 `?page=1&limit=20` 같은 문자열 파라미터
- When: 파라미터를 Zod가 `SearchPartnerParams` 로 파싱함
- Then: `page` 와 `limit` 등이 안전하게 Number형으로 형변환되어, 서비스 로직(Prisma Query)에 직접 주입 가능한 상태여야 한다.

Scenario 2: 배열 파라미터 변환
- Given: `?tags=welding,molding&region=seoul` 파라미터
- When: 파싱 동작을 구동함
- Then: `tags`는 쉼표를 기준으로 문자열 배열 `["welding", "molding"]` 로 안전하게 파싱됨이 명세에 반영되어 있어야 한다.

## :gear: Technical & Non-Functional Constraints
