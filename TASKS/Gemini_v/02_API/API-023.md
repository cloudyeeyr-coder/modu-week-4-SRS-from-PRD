## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: 슬롯 0건에 대한 우회 리턴(추천 일정)
- Given: 클라이언트가 가득 찬 일정(`timeSlot`)을 예약 찌름
- When: 예약 Server Action이 응답함
- Then: 에러코드(`SLOT_NOT_AVAILABLE`)와 함께, 프론트(FC-026)에서 쓸 수 있게 "해당 주차에서 가장 가까운 가용 슬롯" 정보를 메타데이터에 담아 반환해 주어야 한다.

Scenario 2: DTO 날짜 변환 제약
- Given: "YYYY-MM-DD" 로 넘어오는 파라미터 리퀘스트 
- When: 서버 액션 폼에서 수신
- Then: 이 DTO 스키마는 런타임오류를 막기 위해 넘어온 문자열을 타겟 DB 포맷인 DateTime 이나 표준 포맷으로 온전히 Type-Cast 하여 통과시킨다.
