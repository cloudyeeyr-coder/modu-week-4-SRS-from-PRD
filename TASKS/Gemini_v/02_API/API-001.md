## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: 올바른 DTO 입력
- Given: Zod 스키마에 정의된 조건에 모두 부합하는 페이로드
- When: 컴포넌트가 `signupBuyer`를 호출함
- Then: Validation을 무사히 통과하여 `success: true` 와 생성될 구조(Type)를 반환한다.

Scenario 2: 사업자번호 중복 대응 규격
- Given: 이미 가입된 사업자번호로 Zod 스키마를 무사히 통과시킨 페이로드
- When: 내부 로직이 DB 레이어 검증에서 실패함
- Then: 클라이언트가 즉각 인지할 수 있는 에러 코드(`AUTH_DUPLICATE_BIZ_NO`)와 UI 메시지가 `{ errors }` 객체에 담겨 반환된다.
