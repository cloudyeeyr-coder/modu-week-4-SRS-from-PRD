## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: 선행 상태 제약 위반
- Given: 계약 상태가 아직 `inspecting` 이거나 `disputed` 인 계약건 데이터
- When: 관리자가 `confirmRelease` DTO를 사용해 Server Action 시도함
- Then: 이 DTO 명세에 의거해 백엔드 계층에서 선행조건 위반(`ESCROW_NOT_APPROVED_YET`) 에러 메시지를 반환하여 조작을 방어한다.

Scenario 2: DTO 반환 
- Given: 관리자가 올바르게 조작함
- When: 서버가 응답함
- Then: 방출 완료 시각과 함께 시스템 플래그를 Return 응답으로 던져주어 프론트 UI(UI-008)가 업데이트 되도록 만든다.
