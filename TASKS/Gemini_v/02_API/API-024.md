## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: JSONB 내부 객체 Validation
- Given: 보고서를 등록하려는 관리자 앱(폼 데이터)
- When: 서버 액션을 찌를 때, `recommendedSiIds` 에 배열이 아닌 문자열이 섞어 들어옴
- Then: 이 복합 JSONB 스키마를 구성하는 Zod 체결 조건에 의해 타입 오류를 발생시키고 400 Bad Request 급의 메시지를 UI로 반환한다.

Scenario 2: 상태 조건(Precondition) 확인 규격
- Given: 이미 `completed` 가 된, 혹은 아직 예약일이 한참 남은 `bookingDate`를 쥔 예약건
- When: 이 액션으로 접수 시도(모의)
- Then: 시스템 설계상 "현재 보고서를 등록할 조건이 아닙니다" 라는 인터페이스 에러를 응답해야 한다.
