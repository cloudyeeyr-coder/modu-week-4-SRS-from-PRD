## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: 배열/복합 객체 데이터 Validation
- Given: 정상적인 회원 정보와 배열 형태의 `capabilityTags` ["Molding", "Welding"]
- When: `signupSiPartner` Server Action이 호출됨
- Then: Zod 검증을 통해 타입 에러 없이 성공적인 파싱이 이루어져야 한다.

Scenario 2: 검토 상태 반환 안내
- Given: 완벽한 SI 회원가입 페이로드
- When: DB 쓰기를 진행할 액션 인터페이스를 호출함
- Then: 반환 DTO에는 단순히 회원가입 성공 플래그만이 아니라 시스템 상 "Admin Review Pending(검토 대기 중)" 임을 프론트엔드가 인지할 수 있는 추가 필드(또는 메시지)가 존재해야 한다.
