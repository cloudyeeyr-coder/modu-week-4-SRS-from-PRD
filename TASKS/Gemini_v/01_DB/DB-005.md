## :test_tube: Acceptance Criteria (BDD/GWT)
Scenario 1: 모델 간 Foreign Key 관계 무결성 생성
- Given: 정의된 `Contract` 스키마 및 연관된 `BuyerCompany`, `SiPartner` 모델
- When: 마이그레이션 스크립트를 적용함
- Then: 참조 무결성(Referential integrity)을 지니는 외래키 제약조건이 DB 엔진 레벨에 정상적으로 생성된다.

Scenario 2: Enum 전이 통제 (설계 검증)
- Given: 에스크로 상태 흐름 엔진
- When: `status` 플래그를 미지정(Invalid) 값으로 변경하려고 시도함
- Then: DB의 모델 제약 위반, 런타임 단계의 Enum Parsing 실패를 반환하여 상태 안정성을 방어한다.

## :gear: Technical & Non-Functional Constraints
- 성능/결함방지: 시스템에서 오가는 금전 계약금(`totalAmount`)은 부동 소수점 오차를 최소화하기 위해 PostgreSQL 배포 환경을 기준 삼아 `Decimal(15,2)` 타입 등의 정밀 규격을 사용할 것을 권장. 로컬 SQLite 환경일 경우 Prisma 엔진의 Fallback 매개 처리를 확인 필요.
