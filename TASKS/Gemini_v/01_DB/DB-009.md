## :white_check_mark: Task Breakdown (실행 계획)

### Phase A: JSON/배열 데이터 우회 전략
- [ ] Postgres의 배열 및 JSONB는 Prisma에서 SQLite 로컬 동시 보장을 위해, `Json` 타입 컬럼 규약으로 일반화.

### Phase B: Prisma 모델 정의
- [ ] `schema.prisma` 내역 정의
  ```prisma
  model SiProfile {
    id                String   @id @default(cuid())
    siPartnerId       String   @unique
    reviewSummary     Json?    // AI 요약 및 사용자 리뷰 풀 유지
    avgRating         Float?
    completedProjects Int      @default(0)
    failedProjects    Int      @default(0)
    capabilityTags    Json?    // Array of string ["스마트팩토리", "로봇비전"]
    updatedAt         DateTime @updatedAt

    siPartner SiPartner @relation(fields: [siPartnerId], references: [id])
  }
  ```

### Phase C: 컴파일 & 적용
- [ ] CLI `format` 무결성 검증
- [ ] `npx prisma migrate dev --name init_si_profile`

**Scenario 1: JSON 제약 통과 검열**
- Given: Json 컬럼 선언
- When: 로컬 SQLite와 배포용 PostgreSQL 양쪽의 엔진 설정 환경에서 `db push` 발생 시
- Then: 각 DB 인터페이스에 맞추어 `TEXT`(SQLite)나 `JSONB`(PG)로 정상 파싱 및 컴파일됨을 검증.
