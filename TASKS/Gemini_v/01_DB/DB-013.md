## :dart: Summary
- 기능명: [DB-013] `PARTNER_PROPOSAL` 테이블 스키마 및 마이그레이션 작성
- 목적: 로봇 제조사가 SI 파트너에게 파트너십 구축을 요구하는 비동기적인 초청(Invitation) 데이터를 보관. 양측 응답 여부에 대한 TTL(Time-To-Live) 관리가 가능하게 데드라인 기능 포함.

## :white_check_mark: Task Breakdown (실행 계획)

### Phase A: 제안 프로세스 모델 설계
- [ ] 제조사↔파트너 간 N:M 연결을 위한 중간 매핑 설계
  ```prisma
  model PartnerProposal {
    id             String   @id @default(cuid())
    manufacturerId String
    siPartnerId    String
    status         String   @default("pending") // pending, accepted, rejected, expired
    deadline       DateTime // Vercel Cron이 만료 감지용으로 쓸 타임스탬프
    createdAt      DateTime @default(now())
    updatedAt      DateTime @updatedAt

    manufacturer Manufacturer @relation(fields: [manufacturerId], references: [id])
    siPartner    SiPartner    @relation(fields: [siPartnerId], references: [id])
  }
  ```
