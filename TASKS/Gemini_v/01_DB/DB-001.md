## :dart: Summary
- 기능명: [DB-001] Prisma ORM 초기 설정 및 SQLite/PostgreSQL 이중 환경 구성
- 목적: 전체 데이터베이스 스키마의 **단일 진실 공급원(SSOT)** 역할을 하는 Prisma ORM을 초기화하고, 로컬 개발 환경(SQLite)과 프로덕션 배포 환경(Supabase PostgreSQL)에서 동일한 `schema.prisma`로 동작할 수 있는 이중 데이터소스 기반을 확립한다. 이 태스크는 모든 DB-00x 테이블 마이그레이션 및 Seed 스크립트의 기반이 되는 **프로젝트 데이터 레이어의 최초 설정** 태스크이다.

## :white_check_mark: Task Breakdown (실행 계획)

### Phase A: Prisma 패키지 설치 및 초기화
- [ ] `prisma` (DevDependency) 및 `@prisma/client` (Runtime Dependency) 패키지 설치
  ```bash
  npm install -D prisma
  npm install @prisma/client
  ```
- [ ] `npx prisma init` 실행하여 `prisma/schema.prisma` 파일 및 `.env` 기본 구조 생성

### Phase B: `schema.prisma` 기본 설정
- [ ] `datasource db` 블록 설정 — `provider = "postgresql"` (배포 기준), `url = env("DATABASE_URL")`
- [ ] `generator client` 블록 설정 — `provider = "prisma-client-js"`, `previewFeatures` 필요 시 명시
- [ ] `schema.prisma` 내 후속 엔티티 추가를 위한 모델 배치 컨벤션 주석 작성 (도메인별 영역 구분)

### Phase C: 환경 변수 및 보안 설정
- [ ] `.env.example` 파일 생성 — `DATABASE_URL`, `DIRECT_URL` 템플릿 제공
  ```dotenv
  # 로컬 개발 (SQLite)
  DATABASE_URL="file:./dev.db"

  # 배포 환경 (Supabase PostgreSQL) — 실제 값은 Vercel Dashboard에서 관리
  # DATABASE_URL="postgresql://[user]:[password]@[host]:[port]/[db]?pgbouncer=true"
  # DIRECT_URL="postgresql://[user]:[password]@[host]:[port]/[db]"
  ```
- [ ] `.env.local` 파일 생성 및 `.gitignore` 등록 확인 (`.env*`, `prisma/dev.db*`)

### Phase D: Connection Pooling 및 배포 환경 전략 수립
- [ ] Prisma Client 인스턴스 싱글턴 패턴 구현 (`lib/prisma.ts`)
  ```typescript
  import { PrismaClient } from '@prisma/client';
  const globalForPrisma = globalThis as unknown as { prisma: PrismaClient };
  export const prisma = globalForPrisma.prisma ?? new PrismaClient();
  if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma;
  ```

### Phase E: 검증 및 README 문서화
- [ ] `npx prisma format` 실행 구문 오류 검증
- [ ] 로컬 DB 셋업 절차 및 마이그레이션 워크플로우 `README.md` 문서화

## :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: 로컬 개발 환경(SQLite) 연결 및 스키마 동기화 검증**
- Given: `.env.local`에 `DATABASE_URL`이 `file:./dev.db`로 설정됨
- When: `npx prisma db push` 실행
- Then: 프로젝트 루트의 `prisma/`에 `dev.db` 생성 완료, Exit code = 0

**Scenario 2: Prisma Client 싱글턴 인스턴스 정상 동작**
- Given: `lib/prisma.ts`에 싱글턴 패턴 구현 완료
- When: Next.js dev 서버가 HMR(Hot Module Replacement) 리로드 수행
- Then: 콘솔에 "warn(prisma-client) Already N Prisma Clients are actively running" 경고 미발생

**Scenario 3: .gitignore 보안 검증**
- Given: `.env.local` 및 `prisma/dev.db` 생성됨
- When: `git status` 실행
- Then: 해당 파일들이 Untracked/Staged 목록에 노출되지 않음
