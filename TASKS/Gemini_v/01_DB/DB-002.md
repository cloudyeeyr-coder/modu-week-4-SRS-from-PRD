## :test_tube: Acceptance Criteria (BDD/GWT)

**Scenario 1: 모델 및 필드 정상 생성 검증**
- Given: 정의 완료된 `BuyerCompany` Prisma 스키마 구조
- When: 로컬 환경에서 `npx prisma db push` 또는 `migrate dev` 실행 시
- Then: DB 내에 `BuyerCompany` 테이블이 명세된 데이터 타입(문자열 길이 유연성, 고유 키 등)과 타임스탬프 필드를 가진 상태로 정상 구축된다. Exit code = 0

**Scenario 2: 데이터 무결성 검증 - 고유(Unique) 제약 동작**
- Given: 플랫폼 DB 내에 `bizRegistrationNo` 값이 `123-45-67890`인 기업 Row가 이미 존재함
- When: Prisma Client 또는 SQL을 사용해 동일한 사업자번호(`123-45-67890`)의 신규 회원가입 Insert를 시도함
- Then: DB 엔진 단에서 고유 키 제약 위반(Unique Constraint Violation) 에러(`P2002`)를 발생시키며 중복 입력 트랜잭션을 강제 거부한다.

## :gear: Technical & Non-Functional Constraints
- **보안 및 확장성 (UUID/CUID):** ID 값은 외부에서 데이터베이스의 볼륨이나 가입순서를 유추하지 못하도록 자동 증가 정수(Autoincrement) 대신 고유 문자열 모델인 `CUID` 규칙을 적용 필수.
- **다중 환경 호환성:** Enum의 경우 SQLite 제약으로 인해 컴파일 오류가 예상되는 경우, `String` 속성으로 정의하고 TypeScript Layer에서 Union Type으로 강제하는 것을 권장.
